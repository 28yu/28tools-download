# PDF→Excel 実装コード

ハンドオフ資料 (`28yu/Revit-Add-ins:Docs/HANDOFF-PDFtoExcel-Tool.md`) に基づく **アンカーテキスト駆動抽出** 方式の実装。

スキーマ仕様: `../../docs/PDF-to-Excel-Schema-v1.md`
方針修正経緯: `../../docs/PDF-to-Excel-Step2-Report.md`

## 設計

罫線情報には依存せず、**テキスト要素の位置** だけで以下を実現:

1. **アンカー識別**: `FG\d+` パターンで梁符号を検出 (compound `FG17,FG18,(FG19)` も対応)
2. **バンド判定**: アンカー Y 座標クラスタリングで「同じ縦位置の梁グループ」を抽出
3. **オフセット駆動セル抽出**: バンドの FG アンカー Y からの既知オフセット (+12=スパン、+131=主筋径、+141=主筋上... +181=あばら筋) と、FG カード中心 X からの a/c/b 位置オフセット (±62) で各値を取得

PDF レイアウト想定:
- 各バンド = 横一列に並ぶ FG カード群 (FG1〜FG6, FG7〜FG11/FG9A, FG12〜FG17,18,19)
- 各カード = 上から下に 「断面図 / スパン / 成D / 幅B / 主筋径 / 主筋上 / 主筋下 / 端あき1 / 端あき2 / あばら筋」 の固定行構成
- カード内 X = 3 サブ位置 (a:左端 / c:中央 / b:右端)、Δx ≈ 62pt

## ファイル

| ファイル | 内容 |
|---|---|
| `extract-rc-beam.mjs` | RC 基礎大梁ページ抽出（PDF2 p.1 想定） |
| `probe-text.mjs` | テキスト要素を Y クラスタごとに表示するデバッグ用ユーティリティ |
| `sample-output-beams.json` | PDF2 p.1 抽出結果サンプル（20 梁エントリ） |

## 実行方法

```bash
cd pdf-to-excel/src
# pdfjs-dist@4 をインストール済みであること
node extract-rc-beam.mjs    # PDF パスは定数で指定
```

## 既知の課題

- 制御文字 `` (US: Unit Separator) が PDF.js のテキスト抽出に混入する場合があり、現実装は前処理で除去している
- compound 梁 (`FG17,FG18,(FG19)`) は 1 行で抽出される。最終 Excel 出力時に 3 行に展開する必要あり (FG18/FG19 は同一配筋データを共有)
- "FG19のみ" 注記は別バンドに紛れ込むため、最終出力時にフィルタ要
