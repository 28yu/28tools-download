# PDF→Excel 変換ツール: Excel スキーマ仕様 v1

> このドキュメントは、Web ツール（PDF→Excel 変換）が出力する Excel ファイルと、Tools28 アドイン（Excel→Revit ファミリタイプ生成）が読み込む Excel ファイルの **共通フォーマット仕様** を定義する。
>
> - 起点ハンドオフ資料: `28yu/Revit-Add-ins:Docs/HANDOFF-PDFtoExcel-Tool.md`
> - 対応スキーマバージョン: `v1`
> - 確定日: 2026-05-20

## 用語

| 用語 | 意味 |
|---|---|
| **位置別配筋** | 1本の梁・柱の長手方向の異なる位置（端部・中央など）で変わる配筋情報。Revit ファミリパラメータとして位置ごとに個別に保持する |
| **TypeName** | Revit ファミリタイプ名。本仕様では **符号と1対1対応** とする（`FG1` `SG1A` 等） |
| **原文列** | パース前の生文字列を保持する補助列。Web 側パース誤りの検証と Tools28 側の再パースに利用 |

## 確定方針

| 項目 | 決定 |
|---|---|
| ワークブック構成 | `Meta` + カテゴリ別 4 シート構成 |
| 配筋格納 | パース後の数値列 ＋ `*_原文` 列で原文も保持 |
| 欠損値 | 列は必ず確保し、値は空セル。Web UI でセル黄色ハイライト警告 |
| TypeName 生成規則 | **符号のみ**（`FG1`、`SG1`、`SG1A`） |
| RC 梁の位置別配筋 | **5 位置固定**（`a`/`b`/`c`/`d`/`e`）を横展開列として保持 |
| S 梁の連スパン | 1 セルに `;` 区切り CSV（例: `8400;7800`） |
| サブタイプ（`SG1A` 等） | そのまま別タイプ（別行）として出力 |
| Tools28 パーサー方式 | **ヘッダー文字列マッチ**（列順は固定しない） |

## ワークブック構成

```
*.xlsx
├ Meta            … 図面メタ情報・スキーマバージョン
├ RC_Column       … RC 柱
├ CFT_Column      … CFT（コンクリート充填鋼管）柱
├ RC_Beam         … RC 梁（基礎大梁含む）
└ S_Beam          … S 造大梁・小梁
```

該当データが無いシートは空でも構わないが、**シート自体は必ず作成する**。Tools28 側はシート存在チェック → 行数 0 ならスキップ、で対応する。

## シート: `Meta`

`key | value` の 2 列構成。

| key | value 例 | 必須 | 説明 |
|---|---|---|---|
| `schema_version` | `v1` | ✅ | 本仕様のバージョン |
| `source_pdf_filename` | `0501_梁リスト.pdf` | ✅ | 元 PDF ファイル名 |
| `drawing_number` | `S-04` | | 図面番号（取得できれば） |
| `drawing_name` | `基礎大梁リスト(2)` | | 図面名 |
| `extracted_at` | `2026-05-20T10:23:14+09:00` | ✅ | 抽出実行時刻（ISO 8601） |
| `extracted_by` | `pdf2excel-web@0.1.0` | ✅ | Web ツール識別子＋バージョン |

## シート: `RC_Beam`

基本 7 列 + 位置別 10 列 × 5 位置 = **57 列**。

### 基本列

| ヘッダー | 型 | 例 | 備考 |
|---|---|---|---|
| `TypeName` | string | `FG1` | 符号と同一 |
| `符号` | string | `FG1` | |
| `スパン_mm` | int | `6450` | カンマ除去・整数化 |
| `幅B_mm` | int | `800` | PDF に無い場合は空 |
| `成D_mm` | int | `1800` | PDF に無い場合は空 |
| `Fc_Nmm2` | int | `30` | コンクリート強度。PDF に無い場合は空 |
| `備考` | string | | 自由記述 |

### 位置別列（`x` = `a` / `b` / `c` / `d` / `e` を順に展開）

| ヘッダー | 型 | 例 | 備考 |
|---|---|---|---|
| `x_主筋径` | string | `D25` | |
| `x_上_1段` | int | `6` | 上端 1 段目の本数 |
| `x_上_2段` | int | `2` | 上端 2 段目の本数。無ければ `0` |
| `x_下_1段` | int | `6` | 下端 1 段目の本数 |
| `x_下_2段` | int | `0` | 下端 2 段目の本数。無ければ `0` |
| `x_あばら径` | string | `D13` | |
| `x_あばら脚数` | int | `2` | `D13[2]@150` の `[2]` |
| `x_あばらピッチ_mm` | int | `150` | `D13[2]@150` の `150` |
| `x_端あき_mm` | int | `3800` | 配筋の切替境界寸法。無ければ空 |

### サンプル行（FG1）

```
TypeName=FG1, 符号=FG1, スパン_mm=6450, 幅B_mm=, 成D_mm=, Fc_Nmm2=,
a_主筋径=D25, a_上_1段=6, a_上_2段=2, a_下_1段=6, a_下_2段=0,

a_原文=D25 6+2/6 D13[2]@150 3800,
b_主筋径=D25, b_上_1段=6, b_上_2段=0, b_下_1段=6, b_下_2段=3, …,
…(c/d/e 同様)…,
備考=
```

## パース規則

### 配筋表記 → 数値列

| 原文パターン | パース結果 |
|---|---|
| `D25 6` | 径=`D25`, 1 段=`6`, 2 段=`0` |
| `D25 6+2` | 径=`D25`, 1 段=`6`, 2 段=`2` |
| `D25 6+2+1` | 径=`D25`, 1 段=`6`, 2 段=`2`, 3 段=`1`（3 段以降は仕様 v1 では非対応、原文のみ保持） |
| `D13[2]@150` | あばら径=`D13`, 脚数=`2`, ピッチ=`150` |
| `D13@100/200` | v1 非対応（原文のみ保持。端部/中央でピッチ変化の意。位置別列に展開済みなのでこの表記は出ない想定） |

### S 梁断面表記 → 数値列

| 原文パターン | パース結果 |
|---|---|
| `SH-600×200×12×16` | 断面形式=`SH`, 成=`600`, 幅=`200`, tw=`12`, tf=`16` |
| `BH-800×400×16×32` | 断面形式=`BH`, 成=`800`, 幅=`400`, tw=`16`, tf=`32` |
| `H-600×200×12×16` | `SH` として扱う（`H-` は圧延 H と等価） |

### 寸法表記の正規化

| 原文 | 正規化後 |
|---|---|
| `6,450` | `6450`（カンマ除去） |
| `500×500` | `500` と `500` に分解 |
| `500x500` | 同上（小文字 x も許容） |
| `-` | 空セル（「上のセルと同じ」または「該当無し」） |

## Tools28 側の読み出し仕様

- シートを **シート名で取得**（`Meta`, `RC_Column`, `CFT_Column`, `RC_Beam`, `S_Beam`）
- 1 行目をヘッダー行として読み、**ヘッダー文字列で列番号を解決**（列順依存しない）
- 必須ヘッダーが欠けている場合はエラー（どのヘッダーが欠けたかをユーザーに提示）
- セル空欄は `null` として扱い、Revit パラメータには値を設定しない（ユーザーが Revit 上で後から手入力）
- `Meta.schema_version` が未対応バージョンの場合は警告して読込続行

## 既知の制限（v1）

- スキャン PDF・ラスタ PDF は非対応（ベクトル PDF のみ）
- 3 段筋以上（`D25 6+2+1`）は数値列に展開せず原文のみ保持
- RC 梁の端部/中央でピッチが変化する `D10@100/200` 表記は、位置別列に展開済みなので原則出ない想定
- 表内の小さな断面図ノイズの除去（HANDOFF B-3）は v1 では非対応
- 結合セル（HANDOFF B-2）の判定は v1 では Web 側の罫線抽出ロジックに委譲
