# 位相（フェーズ）機能で既存・新設・解体を管理する

Revitのフェーズ機能を使うと、既存建物の改修・増築プロジェクトで「既存」「解体」「新設」の状態を1つのモデルで管理できます。図面セットをフェーズ別に出し分けたり、工事段階ごとの数量集計を行ったりする実践的な方法を解説します。

---

## この記事でわかること

- フェーズとは何か、どんな場面で使うか
- フェーズの作成・名称変更の手順
- 要素に「作成フェーズ」「解体フェーズ」を設定する方法
- フェーズフィルターで既存・新設・解体の表示を切り替える
- フェーズグラフィックスオーバーライドで見た目を制御する
- 改修プロジェクトの典型的なワークフロー

---

## フェーズとは

Revitの**フェーズ（Phase）**は、プロジェクトの時間軸上の「段階」を表す概念です。各要素は「どのフェーズに作成されたか（作成フェーズ）」と「どのフェーズに解体されたか（解体フェーズ）」の2つの属性を持ちます。

ビューにはそれぞれ「ビューフェーズ」と「フェーズフィルター」が設定され、この組み合わせで**何を表示するか**が決まります。

| 要素の状態 | 条件 |
|-----------|------|
| 既存（Existing） | 作成フェーズ ＜ ビューフェーズ かつ 解体フェーズなし |
| 解体（Demolished） | 作成フェーズ ＜ ビューフェーズ かつ 解体フェーズ ＝ ビューフェーズ |
| 新設（New） | 作成フェーズ ＝ ビューフェーズ |
| 将来（Future） | 作成フェーズ ＞ ビューフェーズ |
| 一時（Temporary） | 作成フェーズ ＝ 解体フェーズ ＝ ビューフェーズ |

---

## 手順

### ステップ1: フェーズを作成・管理する

#### 1-1. フェーズダイアログを開く

「管理」タブ → 「フェーズ」パネル → 「フェーズ」をクリックします。

「プロジェクトフェーズ」ダイアログが開き、現在のフェーズ一覧が表示されます。新規プロジェクトには「既存の構築」と「新設構築」の2つがデフォルトで存在します。

#### 1-2. フェーズを追加・名称変更する

| 操作 | 手順 |
|------|------|
| フェーズの追加 | 「前に挿入」または「後に挿入」ボタンをクリック |
| 名称の変更 | フェーズ名をクリックして直接入力 |
| 順序の変更 | 行を選択して「上に移動」「下に移動」で並べ替え |
| フェーズの削除 | 行を選択して「削除」（要素が含まれない場合のみ可能） |

**改修プロジェクトの典型的なフェーズ構成例**:

| フェーズ名 | 用途 |
|-----------|------|
| 既存の構築 | 現況（調査時点の建物） |
| 解体工事 | 解体する既存要素を管理するフェーズ |
| 新設構築 | 新規に追加・建替える要素 |

> **ポイント**: フェーズは時系列順に上から下へ並べます。後から追加するフェーズは必ず適切な位置に挿入してください。一度要素が割り当てられると順序変更の影響が出るため、フェーズ構成はプロジェクト初期に決定するのが理想です。

---

### ステップ2: 要素にフェーズを割り当てる

#### 2-1. 作成フェーズと解体フェーズ

要素を選択すると、「プロパティ」パレットの「フェーズ設定」グループに以下が表示されます。

| プロパティ | 説明 |
|-----------|------|
| 作成フェーズ | この要素が存在し始めるフェーズ |
| 解体フェーズ | この要素が撤去されるフェーズ（設定しない場合は「なし」） |

#### 2-2. 既存要素のモデリング方法

改修プロジェクトで既存建物をモデリングする場合は以下のどちらかの方法を取ります。

**方法A: 作成フェーズを「既存の構築」に設定する（推奨）**

1. 既存壁・床・柱などをモデリングする
2. すべて選択して「プロパティ」→「作成フェーズ」を「既存の構築」に設定
3. 解体フェーズは「なし」のまま

**方法B: リンクされたRevitモデルを既存として使う**

- 既存建物を別の `.rvt` ファイルで受け取る場合は「挿入」タブ → 「Revitのリンク」でリンクし、「管理リンク」からフェーズマッピングを設定します（※Revit 2020以降で精度が向上）。

#### 2-3. 解体要素の設定

取り壊す要素を選択して「解体フェーズ」を対象フェーズに設定します。

> **ショートカット**: 解体ツールを使うとまとめて設定できます。  
> 「修正」タブ → 「フェーズ」パネル → 「解体」をクリック → 解体したい要素を選択。  
> このツールは現在のビューフェーズを解体フェーズとして自動的に割り当てます。

---

### ステップ3: ビューのフェーズ設定を行う

#### 3-1. ビューフェーズを設定する

ビューを開いた状態で「プロパティ」パレットを確認します。

| プロパティ | 説明 |
|-----------|------|
| フェーズフィルター | このビューで要素をどう表示するかのルール |
| フェーズ | このビューが参照するフェーズ（ビューフェーズ） |

**ビューフェーズの設定例（改修プロジェクトの場合）**:

| ビュー名 | ビューフェーズ | フェーズフィルター |
|---------|-------------|----------------|
| 現況平面図 1F | 既存の構築 | 既存すべて表示 |
| 解体平面図 1F | 新設構築 | 解体のみ表示 |
| 新設平面図 1F | 新設構築 | 新設のみ表示 |
| 竣工平面図 1F | 新設構築 | 新設＋既存 表示 |

#### 3-2. フェーズフィルターの設定

「管理」タブ → 「フェーズ」パネル → 「フェーズ」→「フェーズフィルター」タブを開きます。

フェーズフィルターは要素の状態（新設・既存・解体・一時・将来）ごとに「表示」「グラフィックスによる上書き」「非表示」のいずれかを指定できます。

**標準フィルターの概要**:

| フィルター名 | 新設 | 既存 | 解体 | 用途 |
|------------|------|------|------|------|
| すべて表示 | 表示 | 表示 | 表示 | 全状態を確認 |
| 既存すべて表示 | 非表示 | 表示 | 表示 | 現況確認 |
| 新設のみ | 表示 | 非表示 | 非表示 | 新設図面 |
| 解体のみ | 非表示 | 非表示 | 表示 | 解体図面 |
| 前の＋新設 | 表示 | 表示 | 非表示 | 竣工図 |

> カスタムフィルターを作成するには「新規作成」ボタンで追加し、各状態の表示ルールを設定します。

---

### ステップ4: フェーズグラフィックスオーバーライドで見た目を設定する

フェーズごとに線色・線種・塗りつぶしパターンを変えることで、図面上で既存・解体・新設が一目でわかるようになります。

「管理」タブ → 「フェーズ」パネル → 「フェーズ」→「グラフィックスオーバーライド」タブを開きます。

**設定項目**:

| 要素の状態 | 設定できるグラフィックス |
|-----------|----------------------|
| 既存 | 投影線・切断線の色・線種・線幅、サーフェスパターン |
| 解体 | 同上 |
| 新設 | 同上（通常はオーバーライドなし ＝ 標準表示） |
| 一時 | 同上 |

**一般的な設定例（日本の建築図面慣行）**:

| 状態 | 線色 | 線種 | 備考 |
|------|------|------|------|
| 既存 | グレー（128,128,128） | 実線 | 薄く表示して新設と区別 |
| 解体 | 赤（255,0,0） | 破線 | 解体範囲を明示 |
| 新設 | 黒（0,0,0） | 実線（標準） | オーバーライドなし |

> **注意**: グラフィックスオーバーライドはフェーズフィルターで「グラフィックスによる上書き」が設定されているビューにのみ反映されます。フィルター設定と合わせて確認してください。

---

### ステップ5: 改修プロジェクトの典型的なワークフロー

以下は既存建物の部分改修を例にした標準的な進め方です。

```
1. フェーズ構成を決定（プロジェクト初期）
   └─ 既存の構築 / 新設構築（必要に応じて中間フェーズを追加）

2. 既存建物をモデリング
   └─ すべての要素の作成フェーズ = 「既存の構築」

3. 解体要素を設定
   └─ 取り壊す壁・床等の解体フェーズ = 「新設構築」
      または「解体」ツールで一括設定

4. 新設要素をモデリング
   └─ 作成フェーズ = 「新設構築」 が自動設定される
      （ビューフェーズが「新設構築」に設定されていれば）

5. ビューを整備
   └─ 現況図・解体図・新設図・竣工図ごとにビューフェーズ＋フィルターを設定

6. 図面出し・数量集計
   └─ スケジュールにもフェーズフィルターが適用される
      「プロパティ」→「フェーズフィルター」でフェーズ別集計が可能
```

#### フェーズ別スケジュール集計

スケジュールにもフェーズ設定が適用されます。

1. スケジュールビューを開く
2. 「プロパティ」パレット → 「フェーズ」「フェーズフィルター」を設定
3. 「新設のみ」フィルターを適用すれば新設要素だけの数量が集計される

> スケジュールのフェーズ設定と図面ビューのフェーズ設定は**独立しています**。スケジュール専用のフェーズビューを作成して管理すると混乱を防げます。詳細は [03-schedules-quantity.md](./03-schedules-quantity.md) を参照してください。

## Tips

- **ビューテンプレートにフェーズ設定を含める**: 「ビュー」タブ → 「ビューテンプレート」でフェーズとフェーズフィルターをテンプレート化することで、同じ種類のビューを量産するときに設定漏れが防げます。
- **アクティブビューのフェーズを確認する習慣**: 新しい要素を配置する前に、ビューフェーズが意図したフェーズになっているか「プロパティ」パレットで確認してください。意図せず既存フェーズに新設要素が作成されると、図面の出し分けが崩れます。
- **フェーズフィルターは共有ライブラリとして管理**: フェーズフィルターの設定はプロジェクトテンプレートに含めると、複数プロジェクトで一貫した図面表現を維持できます。
- **リンクモデルのフェーズマッピング**: リンクされたRevitファイルの要素には、ホストモデルのフェーズとは別にフェーズマッピングが必要です。「管理リンク」→「フェーズマッピング」タブで対応を設定してください（※Revit 2020以降）。
- **IFC書き出し時のフェーズ対応**: IFCエクスポートではフェーズ別に書き出すことが可能です。「ファイル」→「エクスポート」→「IFC」→「設定」で書き出し対象フェーズを選択できます。

## 注意事項

- **フェーズを削除するとその要素は「既存の構築」に移動される**: フェーズに要素が割り当てられている状態でそのフェーズを削除しようとすると、Revitが警告を表示し、要素は最初のフェーズに割り当て直されます。意図しない変更を防ぐために削除前に必ず確認してください。
- **フェーズ順序の変更は慎重に**: 要素の「既存・新設・解体」判定はフェーズの相対的な順序に依存するため、後から順序を変えると図面の表示が大きく変わることがあります。
- **解体ツールはアクティブビューのフェーズを参照する**: 解体ツールで設定される解体フェーズは「現在開いているビューのフェーズ」です。意図したフェーズのビューで操作しているか必ず確認してください。
- **ワークセット環境でのフェーズ編集**: ワークセットを使う共同作業環境では、フェーズ設定の変更は「プロジェクト標準」として扱われます。中央ファイルに同期するまで他のメンバーには反映されないため、フェーズ構成変更後はすぐに同期することを推奨します。詳細は [01-workset-collaboration.md](./01-workset-collaboration.md) を参照してください。
- **コピー/監視とフェーズの組み合わせ**: コピー/監視した要素は元の要素のフェーズを引き継ぎます。リンクモデルから要素をコピー/監視する際は、作成フェーズが正しく設定されているか確認してください。
