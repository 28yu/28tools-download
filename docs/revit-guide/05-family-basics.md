# ファミリとは何か・挿入方法

Revit の「ファミリ」は、壁・ドア・家具・設備機器などあらゆるモデル要素の基本単位です。ファミリを理解することで、既存のコンポーネントを再利用したり、プロジェクト独自の部品を追加したりできるようになります。この記事ではファミリの種類、ロード・配置・編集の基本操作を解説します。

---

## この記事でわかること

- システムファミリ・ロードファミリ・インプレイスファミリの違い
- ファミリをプロジェクトにロードする方法（ローカルファイル・Autodesk ライブラリ）
- ファミリをモデルに配置する手順
- タイプとインスタンスの違いと使い分け
- インスタンスプロパティ・タイププロパティの編集方法

---

## 手順

### ステップ1: ファミリの3種類を理解する

Revit のファミリは大きく次の3種類に分かれます。どの種類かによって操作方法が変わります。

#### システムファミリ
Revit にあらかじめ組み込まれているファミリです。外部ファイル（`.rfa`）は存在せず、プロジェクト内でのみ管理されます。

- 代表例：壁、床、天井、屋根、階段、手すり、カーテンウォール
- **新規タイプの追加方法**：既存要素を選択 → 「プロパティ」 → 「タイプを編集」 → 「複製」で新しいタイプとして保存します。

#### ロード可能ファミリ（コンポーネントファミリ）
`.rfa` 拡張子の外部ファイルとして保存されており、プロジェクトに読み込んで使用します。カスタマイズの自由度が高く、Revit ファミリの中心的な存在です。

- 代表例：ドア、窓、家具、照明器具、設備機器、構造柱・梁
- 外部から `.rfa` ファイルをロードして使用します（ステップ2で解説）。

#### インプレイスファミリ
プロジェクト内でのみ使用する一品物の形状を作成するためのファミリです。他プロジェクトへの流用はできません。

- 代表例：敷地の特殊な形状、プロジェクト固有の特殊建具
- **作成方法**：「建築（または構造）」タブ → 「モデル」パネル → 「コンポーネント」ドロップダウン → 「インプレイスを作成」を選択します。
- タイプは1つだけ作成できます（複数タイプは作れません）。

---

### ステップ2: ロード可能ファミリをプロジェクトにロードする

#### 方法A：ローカルファイル（.rfa）からロードする

1. **「挿入」タブ**を開き、**「ライブラリからロード」パネル**の**「ファミリをロード」**をクリックします。
2. ファイル選択ダイアログが開きます。ロードしたい `.rfa` ファイルを探してクリックします。
3. **「開く」**をクリックすると、ファミリがプロジェクトにロードされます。
4. ロードされたファミリは**プロジェクトブラウザ**の「ファミリ」ツリーに追加されます。

#### 方法B：Revit 付属のファミリライブラリからロードする

Revit インストール時にファミリライブラリが一緒にインストールされます。

1. 「ファミリをロード」ダイアログを開きます（上記と同じ手順）。
2. **既定の保存場所**（例：`C:\ProgramData\Autodesk\RVT 202X\Libraries\Japan\` など）にアクセスします。
3. カテゴリフォルダ（「建具」「家具」「構造」など）を開いて対象のファミリを選択します。
4. **「開く」**でロードします。

> ※ インストール時にライブラリをインストールしなかった場合は、Autodesk アカウント（manage.autodesk.com）の「製品とサービス」から「ライブラリ」をダウンロードして追加できます。

#### 方法C：ドラッグ＆ドロップでロードする

エクスプローラから `.rfa` ファイルを Revit のビューウィンドウに直接ドラッグ＆ドロップすることでもロードできます。ロードと同時にカーソルにファミリがアタッチされ、そのまま配置できます。

---

### ステップ3: ファミリをモデルに配置する

ロード済みのファミリをモデルに配置する手順です。

1. **「建築」タブ**（または「構造」タブ）を開き、配置したい要素のボタンをクリックします。
   - ドアなら「ドア」、窓なら「窓」、家具なら「コンポーネント」を選択します。
2. **「プロパティ」パレット**のタイプセレクタから、配置したいタイプを選びます。
3. ビュー上の配置したい位置でクリックして確定します。
   - ドア・窓など「ホスト要素」が必要なファミリは、壁など対応する要素の上でないと配置できません。
4. **Esc キー**を押して配置モードを終了します。

> プロジェクトブラウザの「ファミリ」 → 該当カテゴリからもドラッグ＆ドロップで配置できます。

---

### ステップ4: タイプとインスタンスの違いを理解する

ファミリには「タイプ」と「インスタンス」という概念があります。この違いを理解することでプロパティ編集の影響範囲を正しく把握できます。

| 区分 | 説明 | 変更の影響範囲 |
|------|------|--------------|
| **タイプ** | ファミリの「品番」のようなもの（例：幅 900mm の木製ドア） | **同じタイプの全要素**に反映される |
| **インスタンス** | モデルに配置された個々の実体（例：1 階廊下の 3 枚目のドア） | **その要素だけ**に反映される |

たとえば「幅 900 のドア」タイプの幅を 1000 に変更すると、プロジェクト内のそのタイプのドア全てが 1000 に変わります。一方、インスタンスプロパティ（コメントや部屋への関連付けなど）は選択した要素のみに適用されます。

---

### ステップ5: プロパティを編集する

#### インスタンスプロパティの編集

1. 配置済みのファミリをクリックして選択します。
2. **「プロパティ」パレット**に「インスタンスプロパティ」が表示されます。
3. 編集したいパラメータ（コメント、マーク、レベルなど）の値を変更します。
4. **Enter キー**を押すか別の場所をクリックして確定します。変更は選択した要素のみに反映されます。

#### タイププロパティの編集

1. 配置済みのファミリを選択します。
2. **「プロパティ」パレット**の**「タイプを編集」**ボタンをクリックします。
3. 「タイプのプロパティ」ダイアログが開きます。
4. 寸法（幅・高さなど）や材質など、変更したいパラメータを編集します。
5. **「OK」**をクリックして確定します。**同じタイプの全ての要素**に変更が反映されます。

> **新しいタイプを追加したい場合**：「タイプのプロパティ」ダイアログの「複製」ボタンをクリックし、新しいタイプ名を入力してから値を変更します。既存のタイプはそのまま維持されます。

## Tips

- よく使うファミリはプロジェクトテンプレート（`.rte` ファイル）にあらかじめ組み込んでおくと、毎回ロードする手間が省けます。
- メーカー各社が無料の Revit ファミリを提供しています。「（メーカー名） Revit ファミリ ダウンロード」で検索すると多数見つかります。
- ファミリをロードする際、同名ファミリがすでにプロジェクトにある場合は「上書きするか」確認が表示されます。バージョン管理に注意してください。
- ロード済みのファミリは「プロジェクトブラウザ」→「ファミリ」ツリーで一覧表示・管理できます。不要なファミリは右クリック →「削除」でプロジェクトから除去できます。
- ※Revit 2021 以降は「ロード可能なファミリを作成」でファミリエディタが開く手順が統合され、インターフェースが整理されています。

---

## 注意事項

- システムファミリ（壁・床など）は `.rfa` ファイルとして存在しないため、外部からロードできません。プロジェクト内でタイプを複製して管理します。
- インプレイスファミリはプロジェクトに固有のため、他プロジェクトへのコピーや再利用には向きません。汎用性が必要な場合はロード可能ファミリとして作成することを推奨します。
- タイプのプロパティ変更は同じタイプの全要素に影響するため、変更前にどの要素が影響を受けるかを確認してください（「タイプを選択」機能で同タイプ要素を一括選択できます）。
- ファミリファイル（`.rfa`）は Revit のバージョンに依存します。新しいバージョンで作成したファミリは古いバージョンの Revit では開けないことがあります。
- 大量のファミリをロードするとファイルサイズが増大します。使用しないファミリはパージ（削除）してファイルを軽量に保つことを推奨します。（「管理」タブ → 「プロジェクト設定」→「未使用をパージ」）
