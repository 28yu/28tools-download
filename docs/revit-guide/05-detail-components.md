# 詳細コンポーネントとディテールラインで図面を仕上げる

Revitの図面は3Dモデルから自動生成されますが、現場で通用する施工図レベルに仕上げるには2D要素の追加が不可欠です。ディテールラインや詳細コンポーネントを使いこなして、モデルを崩さずに図面品質を高める方法を解説します。

---

## この記事でわかること

- モデル要素と2D詳細要素の違いと使い分け
- ディテールライン・塗りつぶし領域・マスク領域の使い方
- 詳細コンポーネントと繰り返し詳細コンポーネントの活用
- 詳細グループによる再利用
- ディテールビューのベストプラクティス

---

## モデル要素 vs. 2D詳細要素

Revitには「モデル空間」と「シート・ビュー固有の2D空間」があります。この違いを理解することが品質の高い図面作成の第一歩です。

| 種類 | 3D空間に存在するか | 他ビューに影響するか | 主な用途 |
|------|:---:|:---:|---------|
| 壁・床・屋根などのモデル要素 | ✓ | ✓ | 建物の実体を表現 |
| ディテールライン | ✗ | ✗ | 補足線・引き出し線・記号 |
| 塗りつぶし領域 | ✗ | ✗ | ハッチング・スロープ記号など |
| マスク領域 | ✗ | ✗ | モデル線を隠す白塗り |
| 詳細コンポーネント | ✗ | ✗ | 断面材料記号・溶接記号など |
| 寸法・注釈テキスト | ✗ | ✗ | 図面情報の明示 |

> **重要**: 2D詳細要素は**ビューに従属**します。そのビューを削除すると、追加した2D要素もすべて失われます。

---

## 手順

### ステップ1: ディテールラインを追加する

ディテールラインは、補助線・参照線・断面の切り込み線など、モデルにない情報を図面上に描くために使います。

1. 詳細を加えたいビュー（平面図・断面図など）を開く
2. 「注釈」タブ → 「詳細」パネル → 「ディテールライン」をクリック
3. 線スタイルを「プロパティ」パレットまたはオプションバーで選択
4. 図面上に線を描画し、`Esc` で終了

**主な線スタイルの用途**:

| 線スタイル | 用途例 |
|-----------|-------|
| 細実線（Thin Lines） | 寸法補助線・ハッチ境界 |
| 中実線（Medium Lines） | 部材輪郭・仕上げ線 |
| 太実線（Wide Lines） | 断面輪郭・強調線 |
| 破線（Dashed） | 隠れ線・仕上がり範囲 |
| 一点鎖線（Dash dot） | 中心線・軸線 |

> **ヒント**: プロジェクト全体で線スタイルを統一するには「管理」タブ → 「追加設定」→ 「線スタイル」でプロジェクト標準として登録します。

---

### ステップ2: 塗りつぶし領域を使う

塗りつぶし領域（Detail Region）はハッチングパターンを2D図面上に追加するツールです。材料断面の表現や、既製品のシンボル塗潰しに活用します。

1. 「注釈」タブ → 「詳細」パネル → 「領域」→「塗りつぶし領域」
2. スケッチモードで境界線を描く（閉じた輪郭が必要）
3. 「プロパティ」パレットで塗りつぶしパターンを選択
4. 「スケッチを完了」（緑のチェックマーク）で確定

**塗りつぶしパターンの種類**:

| タイプ | 説明 |
|--------|------|
| 製図パターン | ビューのスケールに連動して線間隔が変わる |
| モデルパターン | 実寸で固定（スケール変更で見た目が変わる） |

> 製図図面では**製図パターン**を使うのが原則です。スケールを変えても視覚的な密度が保たれます。

---

### ステップ3: マスク領域で不要な線を隠す

マスク領域は、モデルが自動生成した線を隠すための白い塗りつぶしです。凡例や詳細図の周囲を整理するときに特に有効です。

1. 「注釈」タブ → 「詳細」パネル → 「領域」→「マスク領域」
2. 隠したいモデル線の上をスケッチで囲む
3. 「スケッチを完了」で確定

> **注意**: マスク領域の境界線には線スタイルを「非表示」または「細実線」に設定してください。境界自体が印刷されることがあります。

---

### ステップ4: 詳細コンポーネントを配置する

詳細コンポーネントは、断面の材料記号・溶接記号・建具記号など、**ライブラリとして管理できる2Dシンボル**です。

1. 「注釈」タブ → 「詳細」パネル → 「コンポーネント」→「詳細コンポーネント」
2. 「プロパティ」パレットでファミリとタイプを選択（またはロード）
3. ビュー上でクリックして配置
4. 必要に応じてプロパティで向きやスケールを調整

**詳細コンポーネントのロード**:

- 「挿入」タブ → 「ファミリをロード」→ `Detail Items` フォルダを参照
- 会社独自の2Dファミリ（.rfa）も同様にロード可能

---

### ステップ5: 繰り返し詳細コンポーネントを使う

同じシンボルを一定間隔で連続配置したい場合（例：断熱材の波線、砂利・栗石の断面表現）は「繰り返し詳細コンポーネント」が便利です。

1. 「注釈」タブ → 「詳細」パネル → 「コンポーネント」→「繰り返し詳細コンポーネント」
2. 始点と終点をクリックして線状に配置
3. 「プロパティ」で使用するコンポーネントファミリと間隔を設定

**プロパティの主な設定項目**:

| プロパティ | 説明 |
|-----------|------|
| 詳細 | 使用する詳細コンポーネントファミリ |
| レイアウト | 固定距離 / 固定数 / 最大間隔 など |
| 間隔 | コンポーネント間の距離 |
| 固定 | 始点・終点・中央のどこを基準にするか |

---

### ステップ6: 詳細グループで再利用する

同じ詳細セット（例：外壁の標準断面）を複数ビューで使い回すには「詳細グループ」を活用します。

**グループの作成**:

1. 再利用したい詳細要素（ディテールライン・コンポーネント・注釈を含む）を選択
2. 「修正」タブ → 「作成」パネル → 「グループを作成」
3. グループ名を入力して確定

**グループの配置**:

1. 「注釈」タブ → 「詳細」パネル → 「コンポーネント」→「詳細グループを配置」
2. 対象グループを選択してビューにクリック配置

> **ヒント**: グループを編集するとプロジェクト内の全インスタンスが更新されます。ビューごとに少しずつ異なる詳細が必要な場合は「グループを分解」してから個別編集してください。

---

### ステップ7: ディテールビューのベストプラクティス

詳細図（ディテールビュー）を効率よく管理するための設定をまとめます。

**ビューの種類と使い分け**:

| ビュータイプ | 特徴 | 用途 |
|-------------|------|------|
| 詳細図（Detail View） | 親ビューの一部を拡大、モデルに連動 | 部位の拡大詳細 |
| 製図ビュー（Drafting View） | モデルと独立した純2Dビュー | 標準詳細図・凡例 |

- **繰り返し使う標準詳細図は「製図ビュー」で作成**し、プロジェクトブラウザで管理します。
- 製図ビューは別プロジェクトへ「挿入」タブ → 「ビューを挿入」でコピー可能です（※Revit 2021以降）。

**ビュースケールの設定**:

詳細図のスケールは「プロパティ」パレットの「ビュースケール」で設定します。スケールを変えるとディテールラインの見た目（線の太さ・テキストサイズ）が連動して変わります。

---

## Tips

- **製図ビューはプロジェクトテンプレートに含める**: 会社標準の断面詳細や凡例を製図ビューとしてテンプレートに保存しておくと、新規プロジェクト立ち上げ時の工数を大幅に削減できます。
- **「ディテールラインを表示/非表示」の活用**: ビューテンプレートで「詳細要素」のカテゴリ表示を制御することで、同じモデルから詳細有り/無しの2種類の図面を出し分けられます。
- **詳細コンポーネントのファミリは共有ライブラリで管理**: BIM360やAccDocの共有ファミリフォルダに格納し、チーム全員が同じシンボルを使えるようにすることを推奨します。
- **マスク領域の優先度**: 複数のマスク領域が重なる場合、後から配置したものが前面になります。意図しない隠れが発生したら「前面に移動」「背面に移動」で調整してください。
- **スケッチ編集モードを活用**: 既存の塗りつぶし領域やマスク領域を選択 → 「スケッチを編集」で境界形状を後から変更できます。

---

## 注意事項

- **2D詳細要素はビューに依存する**: ビューを削除・置換すると詳細要素も消えます。ビュー整理の前に必ず確認してください。
- **詳細グループのネストは避ける**: グループの中にグループを入れると、編集・分解時に予期しない挙動が起きやすくなります。
- **繰り返し詳細コンポーネントは印刷スケールに注意**: スケールによってはコンポーネントが密集して読めなくなる場合があります。プリントプレビューで確認してから確定させてください。
- **製図パターンとモデルパターンの混在**: 同一ビュー内で両者を混在させると、スケール変更時に意図しない見た目の変化が生じます。ビュー内で統一することを推奨します。
- **DWGエクスポート時の挙動**: 詳細コンポーネントはDWGエクスポート時にブロック参照に変換されます。DWGの受け取り側で文字化けが発生する場合は、エクスポート設定で「詳細要素をブロックに変換しない」を選択してください（※Revit 2022以降でオプションあり）。
