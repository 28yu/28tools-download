# Excel との連携でデータを効率管理する

Revit の集計表と Excel を連携させることで、部屋名・建具番号・仕上情報などを大量一括編集できます。発注者からもらった室名リストの反映、建具表の番号振り直し、設計変更ごとのパラメータ更新——これらを手作業でやると転記ミスが頻発しますが、正しく連携すれば 1 ファイルの編集が数百の要素に一括反映されます。この記事では、標準機能によるエクスポートから Dynamo を使ったバルクインポートまで、実務で使えるワークフローを具体的に解説します。

---

## この記事でわかること

- 集計表を Excel にエクスポートする正しい手順
- Excel で編集したデータを Revit に戻す方法（手動 / Dynamo）
- 部屋名・建具番号の一括管理フロー（日本の実務に即した例）
- Dynamo スクリプトのノード構成と Null 値の処理方法
- サードパーティツール（Ideate BIMLink 等）の位置づけ
- ラウンドトリップ時の実務でよくある失敗と回避策

## 手順

### ステップ1: 集計表を Excel にエクスポートする

#### 1-1. エクスポートに必要な準備

エクスポート前に集計表の設定を確認する：

- **ElementId（Id）列を追加しておく**: 「フィールド」タブで `Id`（ElementId）を追加する。後で Dynamo を使ってデータを書き戻す際の照合キーになるため、エクスポート前に必ず追加しておく。
- **フィルターを確認する**: フィルターがかかっている集計表はフィルターで非表示になっている行がエクスポートされない。全データが必要な場合はフィルターを一時的に解除してからエクスポートする。
- **「すべてのインスタンスをアイテム化する」をオンにする**: タイプ集計（まとめ表示）の状態だと ElementId が 1 つしか出力されない。往復ワークフローで使う場合は必ずインスタンス単位で表示する。

#### 1-2. テキスト形式でエクスポートする

1. 対象の集計表ビューを開いた状態で「**ファイル**」→「**エクスポート**」→「**レポート**」→「**明細書**」。
2. 保存場所とファイル名を指定（拡張子 `.txt`）。
3. エクスポートオプション：

| オプション | 推奨設定 | 理由 |
|---|---|---|
| タイトルの行数を含む | 1 行 | Dynamo での行番号指定に合わせやすい |
| 列ヘッダーを含む | チェックあり | フィールド名をキーとして使うため |
| テキスト修飾子 | `"` | カンマを含む値（建物名等）の誤分割を防ぐ |
| 区切り文字 | タブ | カンマと全角文字の混在時に安全 |

#### 1-3. Excel でファイルを開く

1. Excel の「**データ**」タブ →「**テキストまたは CSV から**」でインポート（ファイルをそのまま開かない）。
2. 文字コードは「**日本語（シフト JIS）**」または「**UTF-8**」を確認して選択する。
3. 部屋番号・建具番号など先頭ゼロがある列のデータ型は「**文字列**」に明示指定する。数値として読み込むと `0101` が `101` になってしまう。

### ステップ2: Excel でデータを編集する

#### 2-1. 絶対に守るルール

| ルール | 理由 |
|---|---|
| **ElementId 列は変更しない** | Revit との照合キーであり、変更するとすべての書き込みが別の要素（最悪は別のプロジェクト）に対して行われる |
| **列の追加・削除・順序変更をしない** | Dynamo で列インデックスを使っている場合、列がずれると誤った値が書き込まれる |
| **新規行を追加しない（Dynamo 手動インポートの場合）** | 既存要素への更新は機能するが、Dynamo での新規部屋・要素の一括作成には別のワークフローが必要 |
| **セル内改行・タブを入れない** | テキスト形式での再エクスポート時に崩れる |

#### 2-2. 典型的な編集作業（日本のプロジェクト例）

**部屋名・番号の管理（発注者書式への対応）**

発注者から「部屋番号は `B101` 形式に統一すること」などの指示が出たとき：

1. Revit の部屋スケジュールを Excel にエクスポート（番号・名前・面積・ElementId を含む）
2. Excel の「番号」列を発注者書式に一括修正（置換・数式で連番生成など）
3. 「名前」列で全角/半角の揺れを統一（`SUBSTITUTE` 関数などで変換）
4. Dynamo で書き戻し

**建具番号の振り直し**

設計変更で建具配置が変わった際：

1. ドア集計表を Excel にエクスポート（マーク・タイプマーク・レベル・ElementId を含む）
2. Excel でレベルと位置をキーに並べ替えて番号を採番（`D-101`, `D-102` ... `D-201` ...）
3. Dynamo でドア要素の「マーク」パラメータに書き戻し

### ステップ3: データを Revit に戻す

#### 方法A: 手動コピー＆ペースト（10件程度まで）

1. Revit で集計表ビューを開く。
2. Excel の更新した列を選択してコピー。
3. Revit 集計表の先頭セルをクリックしてペースト。

Revit 2023 以降は複数列の一括ペーストが改善されているが、基本的に**同一列・連続する複数行**の操作に限られる。行数が多い場合や列をまたぐ場合は Dynamo を使う。

#### 方法B: Dynamo を使ったバルクインポート（実務の主流）

Dynamo を使うと ElementId で照合しながら一括書き込みできる。

**基本的なノード構成**

```
[File Path]  ← Excelファイルのパスを指定
    ↓
[Data.ImportExcel]  ← シート名 "Sheet1"、startRow=1（ヘッダー行をスキップ）
    ↓
[List.Transpose]  ← 行列を転置して列ごとのリストに変換
    ↓
[List.GetItemAtIndex]  ← 0列目：ElementId列、N列目：書き込む値の列
    ↓
[Element.GetElementIdFromString または Element.SetParameterValueByName]
    ↓
[Element.SetParameterValueByName]  ← パラメータ名と値を書き込む
```

**Null 値の処理（重要）**

Excel のセルが空白の場合、Dynamo は `null` として読み込む。`null` を `SetParameterValueByName` に渡すとエラーになったり既存値が消えたりする。以下のノードで null を空文字列に変換してから書き込む：

```
[Object.IsNull]  ← nullかどうかを判定
    ↓
[List.ReplaceItemAtIndex または IF ノード]  ← nullを "" に置換
    ↓
[Element.SetParameterValueByName]
```

**具体的な手順**

1. 「**管理**」タブ →「**Dynamo**」を起動（またはプロジェクトブラウザの「Dynamo Scripts」から）。
2. 新規グラフを作成し、上記のノードを接続する。
3. `Data.ImportExcel` の `readAsStrings` 入力を `true` にする。数値として読み込むと「101」が `101.0` になり文字列パラメータとの型不一致が起きる。
4. 「**実行**」をクリックして書き込みを実行。
5. Revit で集計表を確認して値が正しく更新されているかを検証する。

> **注意**: Dynamo の実行は `Ctrl+Z` でのアンドゥ対象外。実行前に必ず Revit ファイルを「名前を付けて保存」して退避版を作る。ワークシェアリング環境では実行前に必ずすべてのローカルを同期させ、他のメンバーが作業中でないことを確認する。

**Dynamo スクリプトの保存と共有**

Revit 2024 以降は Dynamo グラフをプロジェクトブラウザの「Dynamo Scripts」に埋め込める。`.dyn` ファイルをプロジェクト内に保存することで、チームメンバー全員が同じスクリプトを使える。ファイルサーバー運用の場合は共有フォルダに `.dyn` を保管し、File Path ノードで参照する方法が一般的。

### ステップ4: 日本の実務に即したワークフロー例

#### 4-1. 部屋名・面積管理フロー（発注者承認用）

```
[Revit] 部屋スケジュール作成
  フィールド: 番号, 名前, 面積, 用途, ElementId
    ↓ TXTエクスポート
[Excel] 発注者書式に整形（番号・名前の列を整理）
    ↓ 発注者に確認・赤入れ
[Excel] 修正後の番号・名前を列に確定
    ↓ Dynamo（SetParameterValueByName: "番号", "名前"）
[Revit] 部屋パラメータが更新
    ↓
[Revit] 平面図のタグ・集計表・仕上表に自動反映
```

> **注意点**: 「面積」は Revit が自動計算する読み取り専用パラメータのため、Excel 側で面積を変更しても Revit には書き戻せない。面積調整が必要な場合は Revit のモデルを修正する。

#### 4-2. 建具番号管理フロー（設計変更対応）

```
[Revit] ドアスケジュール作成
  フィールド: マーク, タイプマーク, レベル, From Room, To Room, ElementId
    ↓ TXTエクスポート
[Excel] レベル・From Room で並べ替えて採番
  例: 1Fドア → D-101〜, 2Fドア → D-201〜
    ↓ Dynamo（SetParameterValueByName: "マーク"）
[Revit] ドアの「マーク」パラメータが更新
    ↓
[Revit] 平面図のドアタグ・建具表に自動反映
```

#### 4-3. 仕様情報の一括入力フロー（施工者向け積算資料対応）

設計後半で施工者からコスト確認のために仕様情報（メーカー・品番・単価）を一括入力するケース：

```
[Excel] 仕様情報一覧表（施工者作成）
  列: ElementId, メーカー, 品番, 単価
    ↓ Dynamo
[Revit] カスタムパラメータ（共有パラメータ）に書き込み
    ↓
[Revit] 集計表で単価 × 数量 の計算値を集計
```

### ステップ5: サードパーティツールの活用

#### Ideate BIMLink

双方向連携に特化した有償プラグイン。GUI でフィールドマッピングを定義し、繰り返しの更新作業を自動化できる。

| 機能 | 内容 |
|---|---|
| マッピング定義の保存・再利用 | 一度設定したフィールド対応を次回から使い回せる |
| 差分のみ更新 | 変更された行だけ Revit に反映（大規模モデルで高速） |
| 部屋の一括作成 | Excel からルームオブジェクトを新規作成できる |
| BIM Collaborate Pro 対応 | クラウド環境でも動作 |

大規模プロジェクト（部屋数 300 以上など）や繰り返しのデータ更新が多い場合に費用対効果が高い。

#### pyRevit（無料・Python ベース）

pyRevit の Excel 連携ツールは複数のスケジュールを一括エクスポートする用途に使える。ただし **pyRevit は Revit 2025 以降では未対応**（2026/05 時点）のため、バージョンを確認してから導入する。

#### Speckle（無料・オープンソース）

Revit-Excel 間の往復だけでなく、他の BIM ツール（Rhino・Grasshopper・Power BI など）との連携もできる。API を通じたリアルタイム同期が可能で、データ管理の規模が大きい場合の選択肢。

## Tips

- **スクリプトをモジュール化して管理する**: 「エクスポート用」「部屋パラメータ更新用」「建具番号更新用」のように目的別に別ファイルに分ける。大きな 1 つのスクリプトは保守が難しくなる。
- **Dynamo の `readAsStrings: true` は基本設定**: 数値として読み込むと `101.0` や `True`/`False` などの型変換が起きて書き込みが失敗することが多い。まず文字列で読み込み、数値が必要な場合のみ `String.ToNumber` で変換する。
- **Excel のバージョン管理**: 更新前の Excel ファイルは日付入りのファイル名でコピーを保持する（例: `部屋リスト_20260510_before.xlsx`）。誤書き込みのロールバック用。
- **「Id」列はシートの先頭列に置く**: Dynamo スクリプトの `List.GetItemAtIndex` でインデックス 0 を常に ElementId に割り当てることで、スクリプトをシンプルに保てる。
- **全角・半角の統一**: 部屋名・建具名で全角/半角が混在すると Revit のフィルターや検索で意図しない挙動になる。Excel で `JIS` 関数や `ASC` 関数を使って統一してから書き戻す。
- **Data.ImportExcel が失敗するとき**: Excel ファイルが開いたままの状態だと `Data.ImportExcel operation failed` エラーになる。Dynamo 実行前に Excel を必ず閉じる。Excel 2013 以上が対象で、古いバージョン（2007 など）は動作しないことがある。

## 実務でよくある失敗

**失敗1: ElementId を間違えて別の要素を上書きした**

ElementId を手入力したり、行をコピーして誤った ID に書き換えたりすると、Dynamo は黙って別の要素のパラメータを上書きする。作業前の「名前を付けて保存」と、書き込み後の集計表サンプル確認を必ず行う。

**失敗2: ワークセット同期後に ElementId が変わった**

ワークシェアリング環境で「新規中央ファイルとして保存」を実行すると、ElementId が採番し直されることがある。これが起きた後に古い Excel データを使って Dynamo を実行すると、存在しない ElementId に書き込もうとしてエラーになるか、間違った要素が更新される。Excel データは中央ファイルの再作成後に必ず再エクスポートし直す。→ [ワークセット運用](./01-workset-collaboration.md) 参照。

**失敗3: 数値パラメータに文字列を書き込もうとした**

「幅」「高さ」などのシステムパラメータは mm 単位の数値型。Excel で「900mm」と入力した値をそのまま書き込もうとすると失敗する。`String.ToNumber` で数値に変換し、Revit の内部単位（フィート：`数値 / 304.8`）に変換して渡す必要がある。

**失敗4: 読み取り専用パラメータに書き込もうとした**

「面積」「周囲長」「体積」など Revit が自動計算するパラメータは Dynamo からも書き込めない。書き込もうとすると `SetParameterValueByName` でエラーが出て、処理がその要素でスキップされる（他の要素には書き込まれる）。対象パラメータが書き込み可能か事前に 1 要素で手動確認する。

**失敗5: Null 処理を忘れて既存値が消えた**

Excel の空白セルを Null チェックなしに `SetParameterValueByName` に渡すと、既存の値が空白で上書きされることがある。`Object.IsNull` で Null を検出し、空文字列に置換してから書き込む。または書き込み対象のリストから Null が含まれるインデックスを `List.FilterByBoolMask` で除外する。

**失敗6: スクリプトを実行したら部屋の「名前」が全部消えた**

インデックスのずれが原因。Excel の列順を変更した後にスクリプトのインデックスを更新し忘れると、たとえば `番号` 列の値が `名前` パラメータに書き込まれる。スクリプト内でカラム名をハードコードするのではなく、ヘッダー行のフィールド名でインデックスを動的に取得するノード構成にすると安全。

## 注意事項

- **Dynamo の実行はアンドゥできない**: `Ctrl+Z` 対象外。実行前に必ずバックアップを取る。
- **ElementId はモデル再作成時に変わる**: 中央ファイルの再作成・新規プロジェクトへの要素コピーなどで変わることがある。Excel データはその都度再エクスポートして使う。
- **ワークシェアリング中は他のメンバーとの競合に注意**: 他のメンバーが作業中のとき Dynamo で一括書き込みをすると、要素の排他ロック（チェックアウト）競合が発生することがある。一括更新は作業時間外かメンバー全員の同意を得てから実行する。
- **共有パラメータとして定義されていないパラメータはタグに出せない**: Excel 経由で入力した値を平面図のタグにも表示したい場合は、そのパラメータが共有パラメータとして定義されているか確認する。→ [共有パラメータの使い分け](./02-shared-parameters.md) 参照。
- **Excel の「自動計算」は書き込み後に混入することがある**: Excel で数式を使って値を計算し、その結果をコピー＆ペーストで値として貼り付けるのを忘れると、Dynamo が読み込む際に数式文字列（`=A2+1` など）がそのまま書き込まれることがある。Excel で「値として貼り付け」してからファイルを保存する。
