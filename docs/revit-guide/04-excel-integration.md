# Excelとの連携でデータを効率管理する

RevitのスケジュールはExcelと連携することで、大量のデータを素早く編集・更新できます。部屋リストやドアリストなど、プロジェクト全体の属性管理に特に効果的です。このガイドでは、エクスポートからインポートまでの往復ワークフローを解説します。

---

## この記事でわかること

- スケジュールをExcelにエクスポートする手順
- Excelで編集したデータをRevitに取り込む方法（手動 / Dynamo）
- Ideate BIMLink などサードパーティツールの概要
- 部屋・ドアデータの一括管理フロー
- ラウンドトリップ時にありがちなミスと回避策

---

## 手順

### ステップ1: スケジュールをExcelにエクスポートする

#### 1-1. スケジュールビューを開く

「プロジェクトブラウザ」でエクスポートしたいスケジュール（例：「部屋スケジュール」）をダブルクリックして開きます。

#### 1-2. テキスト形式でエクスポートする

| 操作 | 説明 |
|------|------|
| 「ファイル」タブ → 「エクスポート」→ 「レポート」→ 「スケジュール」 | 区切り文字付きテキスト（.txt）として書き出す |
| 保存ダイアログで区切り文字を選択 | タブ区切りを推奨（カンマはデータ内に含まれることがある） |

エクスポートされるのは**表示されているフィールドのみ**です。必要な列がスケジュールに含まれているか事前に確認してください。

#### 1-3. ExcelでTXTファイルを開く

1. Excelを起動し「データ」タブ → 「テキストファイル」を選択
2. 区切り文字の種類（タブ）を指定してインポート
3. 各列のデータ型（文字列 / 数値）を確認する

> **ポイント**: 部屋番号やドアタグなどは「文字列」として読み込むと、先頭ゼロが失われません。

---

### ステップ2: Excelでデータを編集する

#### 2-1. 編集上の基本ルール

- **ElementId 列は絶対に変更しない**。これがRevitとの照合キーになります。
- 新規行の追加はRevitへの直接インポートでは機能しません（後述）。
- セル内改行やカンマを含む値は、Revit側の表示が崩れる場合があります。

#### 2-2. 想定される一般的な編集作業

| 用途 | 編集対象の例 |
|------|------------|
| 部屋名の一括変更 | 「部屋名」列をオートフィルや置換で更新 |
| ドアマーク番号の振り直し | 「マーク」列を連番で上書き |
| コメント・メモの一括入力 | カスタムパラメータ列に説明文を追加 |

---

### ステップ3: データをRevitに戻す

#### 方法A: 手動コピー＆ペースト（少量データ向け）

1. Revitでスケジュールビューを開く
2. Excelで更新した値を選択してコピー
3. Revitスケジュール上の対象セルをクリックしてペースト

> **制限**: 複数セルの一括ペーストは行方向にのみ機能します。列をまたぐペーストは非対応です（※Revit 2023以降で一部改善）。

#### 方法B: Dynamoを使ったバルクインポート

Dynamoを使うと、ExcelのデータをElementIdで照合しながら一括書き込みできます。

**基本的なノード構成**:

```
File Path（Excelファイルのパス）
    ↓
Data.ImportExcel（シート名・行番号を指定）
    ↓
List.GetItemAtIndex（ElementId列 / 値列を分離）
    ↓
Element.GetParameterValueByName（既存値確認、任意）
    ↓
Element.SetParameterValueByName（値の書き込み）
```

1. 「管理」タブ → 「ビジュアルプログラミング」→ 「Dynamo」を起動
2. 新規グラフを作成し、上記のノードを順番に接続
3. 「実行」をクリックして書き込みを実行

> **注意**: `Element.SetParameterValueByName` は読み取り専用パラメータ（「フロア」「レベル」等）には書き込めません。対象パラメータが編集可能か事前に確認してください。

#### 方法C: Ideate BIMLink（サードパーティ）

[Ideate BIMLink](https://ideatesoftware.com/ideatabimlink) は、RevitとExcelの双方向連携に特化した有償プラグインです。

| 機能 | 内容 |
|------|------|
| マッピング定義の保存 | 一度設定したフィールド対応を再利用可能 |
| 新規要素の追加 | 部屋やスペースをExcelから一括作成 |
| 差分のみ更新 | 変更があった行だけRevitに反映 |
| BIM 360 / Revit Server対応 | クラウド環境でも動作 |

> 無料トライアルあり。大規模プロジェクトや繰り返しのデータ更新が多い場合に費用対効果が高い。

---

### ステップ4: 部屋・ドアデータの管理フロー

以下はプロジェクトの竣工図対応を例にした典型的なワークフローです。

```
[Revit] 部屋スケジュール作成
    ↓ TXTエクスポート
[Excel] 部屋名・番号・面積を発注者書式に整形
    ↓ 確認・赤入れ
[Excel] 修正した部屋名・番号を列に戻す
    ↓ Dynamo または 手動ペースト
[Revit] 部屋パラメータが更新される
    ↓
[Revit] 平面図・スケジュールに自動反映
```

---

## Tips

- **スケジュールにElementIdを列として追加しておく**: 「スケジュールプロパティ」→「フィールド」で `Id`（ElementId）を追加。Dynamoとのマッピング照合が確実になります。
- **エクスポート前にフィルタを確認**: スケジュールにフィルタがかかっている場合、表示されていない行はエクスポートされません。全データが必要なときはフィルタを一時的に解除します。
- **Dynamoグラフは `.dyn` ファイルとしてプロジェクトに保存**: 「プロジェクトブラウザ」→「Dynamo Scripts」に追加すれば、チームメンバーも同じグラフを使用できます（※Revit 2024以降）。
- **Excelのバージョン管理**: 更新前のExcelファイルはコピーを保持しておく。Revitへの誤書き込みをロールバックできます。
- **共有パラメータを使う**: プロジェクトパラメータではなく共有パラメータとして定義すると、スケジュールへの追加・Dynamoからの参照がしやすくなります。詳細は [02-shared-parameters.md](./02-shared-parameters.md) を参照。

---

## 注意事項

- **読み取り専用パラメータへの書き込みは失敗する**: 「面積」「周囲長」などRevit計算値はDynamoからでも上書きできません。
- **ElementIdはモデルを共有したら変わることがある**: ワークセット同期や新規中央ファイル作成時にElementIdが変わる場合があります。同期直前・直後のExcelデータは照合に使わないこと。詳細は [01-workset-collaboration.md](./01-workset-collaboration.md) を参照。
- **Dynamo実行は元に戻せない**: `Ctrl+Z` でのアンドゥ対象外です。実行前に必ずRevitファイルをバックアップするか、「名前を付けて保存」で退避版を作成してください。
- **Excel側の書式に注意**: 数値として保存されているはずの「面積」をExcelが日付と誤認識することがあります。列の書式を「数値」に明示的に設定してください。
- **大文字・小文字と全角・半角の混在**: 部屋名などで全角/半角が混在すると、Revitのスケジュール集計やフィルタで意図しない挙動になります。Excel上で統一しておくことを推奨します。
