# 共有パラメータとプロジェクトパラメータの使い分け

Revit のパラメータには「共有パラメータ」と「プロジェクトパラメータ」の 2 種類があり、用途や目的によって使い分ける必要があります。違いを理解せずに設定すると、集計表でデータが取れなかったり、タグに表示できなかったりといった問題が発生します。この記事では両者の違いと正しい使い方を解説します。

---

## この記事でわかること

- プロジェクトパラメータと共有パラメータそれぞれの定義と特徴
- どちらを使うべきかの判断基準
- 共有パラメータファイル（`.txt`）の作成手順
- ファミリと集計表へのパラメータ追加方法
- タグで共有パラメータを表示する方法
- よくあるミスと回避策

---

## プロジェクトパラメータと共有パラメータの違い

| 比較項目 | プロジェクトパラメータ | 共有パラメータ |
|---|---|---|
| 定義場所 | プロジェクトファイル（`.rvt`）内 | 外部テキストファイル（`.txt`）内 |
| ファミリへの追加 | 不可 | 可能 |
| タグへの表示 | 不可 | 可能 |
| 集計表への表示 | 可能 | 可能 |
| 複数プロジェクト間の共有 | 不可（プロジェクト固有） | 可能（同じ`.txt`ファイルを参照） |
| IFC エクスポート時の扱い | カスタム属性として出力 | カスタム属性として出力（GUIDで識別） |
| 作成の手間 | 少ない | やや多い（外部ファイルの管理が必要） |

**判断基準のまとめ**:

- タグに表示したい → **共有パラメータ**一択
- ファミリのパラメータとして使いたい → **共有パラメータ**一択
- 複数プロジェクトで同じパラメータを使い回したい → **共有パラメータ**
- そのプロジェクトだけで使う集計用データを追加したい → **プロジェクトパラメータ**で十分

---

## 手順

### ステップ1: 共有パラメータファイルを作成する

共有パラメータは外部のテキストファイル（`.txt`）に定義を保存します。このファイルはチームで共有フォルダに置いて管理します。

1. 「**管理**」タブ → 「**設定**」パネル → 「**共有パラメータ**」をクリック。
2. 「共有パラメータファイルの編集」ダイアログが開く。
3. 「**作成...**」ボタンをクリックし、保存場所とファイル名を指定（例: `SharedParameters_ProjectABC.txt`）。  
   ※ チームで使う場合は社内サーバーや BIM 360 のプロジェクトフォルダに保存する。
4. 「**グループ**」枠内の「**新規**」をクリックし、グループ名を入力（例: `建築情報`、`設備情報`）。  
   グループはパラメータを論理的に分類するための単位。
5. 「**パラメータ**」枠内の「**新規**」をクリックし、以下を設定する：

   | 設定項目 | 説明 |
   |---|---|
   | 名前 | パラメータの名前（例: `製品番号`、`施工区分`） |
   | 規律 | 「共通」を選ぶことが多い。設備系は「HVAC」などを選ぶ場合も |
   | パラメータのタイプ | テキスト・整数・長さ・面積・Yes/No など |

6. 「OK」で保存。

> **ファイル管理のポイント**: 共有パラメータファイルは一度 GUID が割り振られると、同じ名前でも別ファイルとして扱われます。ファイルを複製・再作成すると GUID が変わり、既存の参照が壊れるため、**ファイルは慎重に管理**してください。

---

### ステップ2: プロジェクトに共有パラメータを追加する

共有パラメータをプロジェクトのカテゴリ（壁・ドア・部屋など）に追加して集計表や要素プロパティで使えるようにします。

1. 「**管理**」タブ → 「**設定**」パネル → 「**プロジェクトパラメータ**」をクリック。
2. 「**追加**」をクリック。
3. 「パラメータのプロパティ」ダイアログで「**共有パラメータ**」を選択し、「**選択**」をクリック。
4. 共有パラメータファイルが読み込まれていない場合は「**参照**」で `.txt` ファイルを指定。
5. 対象のグループとパラメータを選んで「OK」。
6. 「パラメータのデータ」セクションで以下を設定：

   | 設定項目 | 説明 |
   |---|---|
   | パラメータのグループ化 | プロパティパレット上の表示グループ（「テキスト」「拘束」など） |
   | カテゴリ | このパラメータを追加するカテゴリ（「ドア」「壁」など。複数選択可） |
   | タイプまたはインスタンス | タイプ属性か、インスタンス個別の属性か |

7. 「OK」で追加完了。

---

### ステップ3: ファミリに共有パラメータを追加する

ファミリ（`.rfa`）に共有パラメータを追加することで、プロジェクトに読み込んだ後も集計表やタグで値を利用できます。

1. 対象ファミリをファミリエディタで開く（`.rfa` ファイルを Revit で開く）。
2. 「**管理**」タブ → 「**設定**」パネル → 「**共有パラメータ**」で、プロジェクトと同じ `.txt` ファイルを参照しているか確認。
3. 「**作成**」タブ（または「**管理**」タブ）→「**プロパティ**」パネル → 「**ファミリタイプ**」をクリック。
4. 「**パラメータを追加**」→「**共有パラメータ**」を選択。
5. 対象のパラメータを選んで「OK」。
6. 必要に応じてパラメータを式や他のパラメータに関連付ける。
7. ファミリを上書き保存し、プロジェクトに「**再読み込み**」する。

> ※ プロジェクトとファミリで**同じ `.txt` ファイルの同じパラメータ**を参照していることが重要です。ファイルが異なると GUID が一致せず、集計表でファミリの値が取得できません。

---

### ステップ4: タグで共有パラメータを表示する

タグファミリに共有パラメータを追加することで、ビュー上に値を表示できます。

1. 対象のタグファミリ（例: `ドアタグ.rfa`）をファミリエディタで開く。
2. 「**作成**」タブ → 「**テキスト**」パネル → 「**ラベル**」を配置（または既存のラベルをダブルクリック）。
3. 「ラベルの編集」ダイアログで「**パラメータを追加**」（右側の「+」ボタン）をクリック。
4. 「**共有パラメータ**」を選択し、ステップ1 で作成したパラメータを指定。
5. 「OK」→ ファミリを保存。
6. プロジェクトに再読み込みすると、タグにパラメータ値が表示される。

> **注意**: タグはそのカテゴリの共有パラメータしか表示できません。ドアタグなら「ドア」カテゴリに追加された共有パラメータのみ選択できます。

---

### ステップ5: プロジェクトパラメータを追加する（シンプルなケース）

タグ表示もファミリへの組み込みも不要で、プロジェクト内の集計だけが目的の場合は、プロジェクトパラメータが手軽です。

1. 「**管理**」タブ → 「**設定**」パネル → 「**プロジェクトパラメータ**」→「**追加**」。
2. 「パラメータのプロパティ」で「**プロジェクトパラメータ**」を選択。
3. 名前・規律・パラメータのタイプを設定。
4. カテゴリを選択して「OK」。

---

### ステップ6: 集計表でパラメータ値を確認・編集する

追加したパラメータは集計表（スケジュール）からまとめて編集できます。

1. 「**ビュー**」タブ → 「**作成**」パネル → 「**明細表**」→「**明細書/数量**」。
2. カテゴリを選択して「OK」。
3. 「フィールド」タブで追加したパラメータを「**使用可能なフィールド**」から「**明細書フィールド**」に追加。
4. 集計表ビューで値を直接入力・編集できる（インスタンスパラメータの場合）。

---

## Tips

- 共有パラメータファイルはバージョン管理ツール（Git など）で管理するか、変更のたびにバックアップを取ること。ファイルが壊れると全プロジェクトの参照が失われる。
- パラメータ名は**英数字・アンダースコアのみ**で構成すると、IFC エクスポートやデータ連携時にトラブルが起きにくい。日本語名も使えるが、外部システムとの連携を考慮する場合は注意。
- 「パラメータのタイプ」は後から変更できないため、設計段階で慎重に決める。後から変更が必要になった場合は削除して新規作成するしかなく、入力済みのデータが失われる。
- ※Revit 2023 以降、「**グローバルパラメータ**」（Global Parameters）を使うと、プロジェクト全体で参照できる変数的なパラメータを設定できる。寸法や値の一括管理に便利。

---

## 注意事項

- 共有パラメータファイルを**複数人が同時に編集**すると競合が発生する。ファイルの編集は担当者を決めて一元管理する。
- プロジェクトに追加したパラメータを削除すると、そのパラメータに入力された**全データが消える**。削除前に集計表などでエクスポートしてバックアップを取ること。
- ファミリにパラメータを追加した後、プロジェクトに再読み込みする際に「パラメータの置き換え」を誤って選択すると既存データが上書きされる場合がある。再読み込み前にプロジェクトを保存しておくこと。
- 共有パラメータを「タイプ」として追加するか「インスタンス」として追加するかによって集計表での扱いが変わる。同じタイプの要素全体に同じ値を設定したい場合はタイプ、個別に値を設定したい場合はインスタンスを選ぶ。
- プロジェクトパラメータは別プロジェクトへ**直接コピーできない**。転用する可能性がある場合は最初から共有パラメータで作成しておく。
