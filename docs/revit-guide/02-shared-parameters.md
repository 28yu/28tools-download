# 共有パラメータとプロジェクトパラメータの使い分け

Revit のパラメータには「共有パラメータ」と「プロジェクトパラメータ」の 2 種類があります。違いを理解せずに設定すると、集計表でデータが取れない・タグに表示できない・IFC 書き出しで属性が消えるといった問題が後から発覚します。この記事では、実務レベルで使える判断基準と、チームでの共有パラメータファイル管理の方法を解説します。

---

## この記事でわかること

- プロジェクトパラメータと共有パラメータの違いと使い分けの判断基準
- 共有パラメータファイル（`.txt`）のチーム管理方法と命名規則
- GUID ベースの同一性確認の仕組みと「同名別物」になる原因
- ファミリ・プロジェクト・タグへの追加手順
- IFC 書き出しと共有パラメータの関係
- 実務でよくある失敗と対処法

---

## プロジェクトパラメータと共有パラメータの違い

| 比較項目 | プロジェクトパラメータ | 共有パラメータ |
|---|---|---|
| 定義場所 | プロジェクトファイル（`.rvt`）内 | 外部テキストファイル（`.txt`）内 |
| ファミリへの追加 | 不可 | 可能 |
| タグへの表示 | 不可 | 可能 |
| 集計表への表示 | 可能 | 可能 |
| 複数プロジェクト間の再利用 | 不可（プロジェクト固有） | 可能（同じ `.txt` ファイルを参照） |
| IFC 書き出し時の属性識別 | GUID なし（内部名のみ） | GUID で確実に識別 |
| 別プロジェクトへのコピー | 不可 | 可能 |
| 作成の手間 | 少ない | やや多い（外部ファイルの管理が必要） |

**判断基準のまとめ**:

- タグに値を表示したい → **共有パラメータ一択**
- ファミリのパラメータとして組み込みたい → **共有パラメータ一択**
- 複数プロジェクトで同じパラメータを使い回したい → **共有パラメータ**
- IFC / COBie での属性連携が必要 → **共有パラメータ**（GUID で確実にマッピングされる）
- そのプロジェクト限定の集計用データだけが目的 → **プロジェクトパラメータ**で十分

---

## 手順

### ステップ1: 会社・プロジェクト用の共有パラメータファイルを設計する

共有パラメータファイルは**チームで一元管理するマスターファイル**です。プロジェクト開始前に BIM 管理者が設計します。

**ファイル管理の基本方針**

- 1 つの会社・部門に対してファイルは原則 1 本（または分野別に 1 本ずつ）に集約する
- ファイルは社内サーバーまたは BIM 360 / ACC の管理フォルダに置き、**全員が同じパスを参照する**
- 編集権限は BIM 管理者のみ。一般メンバーは読み取り専用でアクセスする
- ファイルを変更したら必ずバックアップを取る（`_YYYYMMDD` サフィックスで世代管理）

**ファイル名・ディレクトリの例**:
```
\\server\BIM_Standards\
    SharedParams_Architecture.txt     ← 意匠系パラメータ
    SharedParams_Structure.txt        ← 構造系パラメータ
    SharedParams_MEP.txt              ← 設備系パラメータ
    _backup\
        SharedParams_Architecture_20260401.txt
        SharedParams_Architecture_20260501.txt
```

**グループとパラメータの命名規則**

パラメータ名は「`分野略称_用途_単位または種別`」の形式が実務で使いやすいです。

```
グループ: 建築_施工管理
  A_施工区分          （テキスト）   例: "RC打放し", "ALC"
  A_仕上材料          （テキスト）   例: "EP塗装"
  A_防火区画          （テキスト）   例: "1時間耐火"

グループ: 建築_プロジェクト管理
  A_部屋番号          （テキスト）   例: "101", "202"
  A_工事区分          （テキスト）   例: "新築", "既存", "解体"
  A_BIM_LOD           （整数）       例: 200, 300, 400

グループ: 設備_機器情報
  M_製品番号          （テキスト）   例: "PAC-12345"
  M_製造メーカー      （テキスト）   例: "三菱電機"
  M_風量_CMH          （数値）       例: 1200

グループ: IFC連携用
  IFC_ObjectType      （テキスト）   IFC の ObjectType マッピング用
  IFC_Description     （テキスト）   IFC の Description 属性用
```

> **日本語パラメータ名について**: Revit 上での表示・集計は問題なく使えます。ただし IFC 書き出しや外部データベースとの連携では文字コードの問題が起きやすいため、外部連携前提のパラメータは英数字・アンダースコアのみを推奨します。

---

### ステップ2: 共有パラメータファイルを作成する

1. 「**管理**」タブ → 「**設定**」パネル → 「**共有パラメータ**」をクリック。
2. 「**作成...**」ボタンで保存場所とファイル名を指定（上で決めたパスに保存）。
3. 「**グループ**」欄の「**新規**」でグループを作成する（例: `建築_施工管理`）。
4. 「**パラメータ**」欄の「**新規**」でパラメータを追加する：

   | 設定項目 | 説明 |
   |---|---|
   | 名前 | 上記命名規則に従った名前 |
   | 規律 | 「共通」を選ぶことが多い。設備系は「HVAC」「電気」など |
   | パラメータのタイプ | テキスト・整数・長さ・面積・Yes/No など。**後から変更不可** |

5. 「OK」で保存。

> **重要: GUID について**
> 共有パラメータファイルを作成した瞬間に、各パラメータには固有の GUID（識別子）が割り振られます。Revit はパラメータの「名前」ではなく **GUID で同一性を判定**します。同じ名前でも別ファイルで作成したパラメータは別物として扱われます。ファイルを複製・再作成すると GUID が変わり、既存ファミリやプロジェクトとの参照が切れます。

---

### ステップ3: プロジェクトに共有パラメータを追加する

1. 「**管理**」タブ → 「**設定**」パネル → 「**プロジェクトパラメータ**」→「**追加**」。
2. 「**共有パラメータ**」を選択し「**選択**」をクリック。
3. 共有パラメータファイルが読み込まれていない場合は「**参照**」でファイルを指定。
4. グループとパラメータを選んで「OK」。
5. 「パラメータのデータ」で以下を設定：

   | 設定項目 | 選択指針 |
   |---|---|
   | パラメータのグループ化 | プロパティパレット上の表示場所。「テキスト」「拘束」「ID データ」など |
   | カテゴリ | 壁・ドア・部屋など。複数選択可 |
   | タイプ or インスタンス | 同タイプの全要素に同じ値 → タイプ。要素ごとに異なる値 → インスタンス |

---

### ステップ4: ファミリに共有パラメータを追加する

ファミリに共有パラメータを追加するとき、**プロジェクトと同じ `.txt` ファイルの同じパラメータ（同一 GUID）を参照**しないと、集計表でファミリの値が取得できません。

1. 対象ファミリ（`.rfa`）をファミリエディタで開く。
2. 「**管理**」→「**共有パラメータ**」で、チームの共有パラメータファイルが設定されているか確認。別のファイルが設定されていたら「**参照**」で正しいファイルに切り替える。
3. 「**作成**」タブ → 「**ファミリタイプ**」→「**パラメータを追加**」→「**共有パラメータ**」。
4. 対象パラメータを選び、タイプ/インスタンスを設定して「OK」。
5. 必要に応じて式や寸法に関連付ける。
6. ファミリを上書き保存し、プロジェクトに「**再読み込み**」する。

**複数のファミリに一括追加する場合**:
「管理」→「プロジェクトパラメータ」から同じパラメータを複数カテゴリに一括追加できます（ファミリ個別に作業するより効率的です）。

---

### ステップ5: タグで共有パラメータを表示する

1. タグファミリ（例: `ドアタグ.rfa`）をファミリエディタで開く。
2. 「**作成**」タブ → 「**ラベル**」を配置（または既存のラベルをダブルクリック）。
3. 「ラベルの編集」で「**パラメータを追加**（+ボタン）」→「**共有パラメータ**」。
4. 対象パラメータを選んで「OK」→ ファミリを保存してプロジェクトに再読み込み。

> **注意**: タグはそのカテゴリに追加された共有パラメータしか表示できません。ドアタグには「ドア」カテゴリに設定された共有パラメータのみ表示されます。

---

### ステップ6: プロジェクトパラメータを追加する（シンプルなケース）

タグ表示やファミリへの組み込みが不要で、プロジェクト内の集計だけが目的であればプロジェクトパラメータが手軽です。

1. 「**管理**」→「**プロジェクトパラメータ**」→「**追加**」。
2. 「**プロジェクトパラメータ**」を選択。
3. 名前・規律・タイプ・カテゴリを設定して「OK」。

> ただし、後でタグ表示が必要になったときや別プロジェクトに転用したくなったときに**共有パラメータへ変換する手段がない**（新規作成して値を入れ直すしかない）ため、迷ったら最初から共有パラメータで作ることを推奨します（Revit Peeler などの逆変換ツールも存在しますが公式サポートはありません）。

---

### ステップ7: 集計表でパラメータ値を一括確認・編集する

1. 「**ビュー**」→「**明細書**」→「**明細書/数量**」でカテゴリを選択して集計表を作成。
2. 「フィールド」タブで追加したパラメータを選択して集計表に表示。
3. 集計表ビューでセルをクリックして直接入力・編集できる（インスタンスパラメータの場合）。

これにより、100 枚のドアの「施工区分」を集計表から一括入力でき、図面を 1 枚 1 枚開く必要がなくなります。

---

## IFC 書き出しと共有パラメータの連携

共有パラメータを IFC で正しく書き出すには、パラメータマッピングファイルの設定が必要です。

- Revit の「IFC 書き出し」ダイアログ →「**ユーザー定義プロパティセット**」でマッピングファイル（`.txt`）を指定
- マッピングファイルで「Revit パラメータ名 → IFC PropertySet 名 + プロパティ名」を定義する

**例（防火区画情報を IFC に含める場合）**:
```
PropertySet:  Pset_WallCommon  I  IfcWall
  FireRating  Text  A_防火区画
```

> IFC 書き出しでは共有パラメータの GUID が使われるため、同一名でも別 GUID のパラメータはマッピングが正しく機能しません。プロジェクトで使う GUID を統一することが IFC 連携品質の鍵です。

## 実務でよくある失敗と対処法

### 失敗1: 集計表にファミリのパラメータ値が表示されない

**症状**: ファミリにパラメータを追加し値を入力したのに、集計表の該当列が空欄のまま。

**原因**: ファミリとプロジェクトで**異なる共有パラメータファイルのパラメータ（別 GUID）** を使っている。見た目は同じ名前でも Revit は別物として扱う。

**対処**:
1. 共有パラメータファイルのテキストを開き（メモ帳等）、ファミリとプロジェクトで参照している PARAM の `GUID` 行が一致しているか確認
2. 不一致なら、ファミリを再度開いてチームのマスターファイルを参照し直し、パラメータを追加してプロジェクトに再読み込みする

### 失敗2: 共有パラメータファイルを誤って削除・移動してしまった

**症状**: 「共有パラメータファイルが見つかりません」と表示され、パラメータの追加ができない。

**原因**: ファイルを移動・削除した、またはサーバーのパスが変更された。

**対処**:
- バックアップから元のファイルを復元し、元のパスに戻す（または全メンバーに新しいパスを通知して再参照させる）
- すでにプロジェクトやファミリに追加済みのパラメータのデータは失われていない（GUID がファイル内に存在するだけなので、ファイルが見つからなくてもデータ本体はモデル内に残っている）
- ただし新規パラメータの追加や編集ができなくなるため、速やかにファイルを復元する

### 失敗3: 「タイプ」で追加すべきところを「インスタンス」で追加してしまった（またはその逆）

**症状**: 同じ建具タイプなのにドアごとに値がバラバラになっている、または全ドアに同じ値しか入れられない。

**原因**: タイプ/インスタンスの選択を間違えた。

**対処**:
- パラメータのタイプ（テキスト・数値など）は変更できないが、タイプ/インスタンスの違いは削除して作り直すしかない
- **削除時にそのパラメータのデータは消える**。削除前に必ず集計表で CSV エクスポートして値をバックアップすること
- 再発防止: パラメータを追加する前に「これは全 600 号室で同じ値か？（タイプ）それとも号室ごとに異なるか？（インスタンス）」を確認してから設定する

### 失敗4: ファミリをプロジェクトに再読み込みしたら既存の値が消えた

**症状**: ファミリを更新して再読み込みしたら、プロジェクトで入力していたパラメータの値がリセットされた。

**原因**: ファミリのパラメータを一度削除して作り直した（GUID が変わったため）、またはパラメータを「置き換え」オプションで上書きした。

**対処**:
- 再読み込み前に必ずプロジェクトを保存（Ctrl + S）
- 問題が発生したら保存前の状態に戻すか、事前にエクスポートした CSV からデータを再投入する
- **パラメータの修正はファミリ内で既存パラメータを削除せず設定を変更する**（削除 → 新規作成は GUID が変わるため避ける）

### 失敗5: プロジェクトパラメータで運用してしまい、後で別プロジェクトに転用できなかった

**症状**: 前プロジェクトで作った施工情報パラメータを新プロジェクトで使いたいが、プロジェクトパラメータなのでコピーできない。

**対処**:
- 新プロジェクトで同じ定義のプロジェクトパラメータを手動で作り直す
- または、この機会にプロジェクトパラメータを廃止し、共有パラメータに切り替える
- 再発防止: 将来の転用可能性が少しでもあるパラメータは最初から共有パラメータで作る

## Tips

- **パラメータのタイプは後から変更できない**。設計時に「テキスト」か「整数」か「Yes/No」かを慎重に決める。特に「施工区分」のようなコード値は、後でデータ集計したくなることを想定してテキストより整数やリストの方が扱いやすい場合もある。
- **グローバルパラメータ（Revit 2016 以降）**: プロジェクト全体で参照できる定数的な変数。「基準フロア高さ 3200mm」のような値を一か所で管理し、他の寸法が参照する用途に便利。共有パラメータとは別の仕組みなので混同しないよう注意。
- **共有パラメータファイルの `.txt` の中身を直接編集してはいけない**。GUID や区切り文字を壊すと Revit が読み込めなくなる。パラメータの追加・変更は必ず Revit の「管理」→「共有パラメータ」から行う。
- **dRofus・BIM 360 Field などのデータ管理ツール**: 共有パラメータと連携できるツールが複数あり、大規模プロジェクトでは部屋データや機器情報を外部 DB と双方向同期できる。

---

## 注意事項

- 共有パラメータファイルを**複数人が同時に編集**すると競合が発生する。編集権限は BIM 管理者に集約し、一元管理する。
- プロジェクトに追加したパラメータを削除すると、**そのパラメータに入力された全データが消える**。削除前に集計表を CSV エクスポートしてバックアップを取ること。
- Revit は 1 プロジェクトにつき「現在の共有パラメータファイル」を 1 本しか設定できないが、参照するファイルを切り替えれば複数ファイルのパラメータを同一プロジェクトに追加できる（切り替え前に追加済みのパラメータのデータは保持される）。
- ファミリを配布・納品する際は、受け取り先の共有パラメータファイルと GUID が一致しているか事前に確認する。不一致の場合は相手のファイルに合わせてパラメータを追加し直す必要がある。
